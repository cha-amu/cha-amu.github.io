# 채아무 UI 구현 문서

## 목적

이 문서는 구현 전에 각 페이지의 화면 구조, 공통 UI, 반응형 동작, 사용 자산을 정리한다.

기술 스택은 이미 확정된 대로 Vite + React + TypeScript를 사용한다. 배포 대상은 GitHub Pages이며, 라우팅은 Vite multi-page app 방식을 우선한다.

## 사용 이미지 자산

현재 UI에서 사용할 이미지 자산은 아래 경로와 이름으로 확정한다.

```txt
docs/assets/ui/cha-amu-logo.png      # 사이트 로고
docs/assets/ui/posts-icon.png        # 아무글 아이콘
docs/assets/ui/guestbook-icon.png    # 방명록 아이콘
docs/assets/ui/archive-icon.png      # 자료 아이콘
docs/assets/ui/search-icon.png       # 검색 아이콘
```

구현 시 실제 앱 자산 위치는 아래처럼 둘 수 있다.

```txt
/public/assets/ui/cha-amu-logo.png
/public/assets/ui/posts-icon.png
/public/assets/ui/guestbook-icon.png
/public/assets/ui/archive-icon.png
/public/assets/ui/search-icon.png
```

문서 자산은 디자인 기준이고, 구현 시 `public/assets/ui/`로 복사해서 사용한다.

## 확정 페이지

```txt
/            홈
/posts/      아무글
/guestbook/  방명록
/archive/    자료
/search/     통합 검색
/admin/      관리자
```

## 전체 디자인 방향

- 밝은 테마만 우선 만든다.
- 모바일 우선은 아니지만 모바일에서도 모든 기능이 동작해야 한다.
- 둥글고 부드러운 분위기.
- 전체 시각 스타일은 두들/손그림 스타일로 한다. 픽셀 스타일이 아니다.
- 로고/아이콘의 손그림 느낌을 해치지 않는 단순한 UI.
- 카드형 UI를 고집하지 않고, 내용에 맞는 UI를 쓴다.
- 과한 애니메이션보다 읽기 쉬움과 조작 안정성을 우선한다.

## 색상

기본 색상은 아래 값으로 확정한다.

```txt
--color-primary: #B7DDBF;      /* 메인 컬러 */
--color-background: #F9F9F9;   /* 전체 배경색 */
--color-danger: #F88A87;       /* 삭제/숨김/금지/실패 같은 부정적 동작 */
```

사용 기준:

- `#B7DDBF`는 주요 버튼, 활성 메뉴, 강조 배경, 선택 상태에 사용한다.
- `#F9F9F9`는 전체 페이지 배경의 기본값으로 사용한다.
- `#F88A87`은 삭제, 숨김 처리, 취소 불가에 가까운 경고성 액션, 저장 실패 같은 위험/부정 상태에 사용한다.
- 삭제 버튼은 색만으로 구분하지 말고 `삭제`, `숨김 처리`, `실패` 같은 텍스트와 함께 표시한다.
- 본문 텍스트, 보조 텍스트, 테두리 색은 위 색과 어울리도록 구현 단계에서 정하되 가독성을 우선한다.

## 시각 스타일

- 스타일 키워드: 두들, 손그림, 부드러움, 단순함.
- 픽셀아트처럼 각진 격자 느낌, 레트로 게임 UI 느낌은 사용하지 않는다.
- 선은 완전히 기계적인 직선보다 약간 느슨한 느낌을 허용한다.
- 버튼/입력창/영역 구분은 너무 딱딱한 박스보다 둥근 모서리와 여백으로 정리한다.
- 아이콘은 현재 제공된 손그림 이미지들과 어울리는 크기와 여백으로 배치한다.



## 레이아웃 시스템

페이지마다 임의의 가로 폭을 쓰지 않는다. 공통 레이아웃 토큰으로 좌우 기준선을 맞춘다.

```txt
--layout-page: 960px;       /* 홈을 제외한 모든 페이지의 외곽 본문 폭 */
--layout-readable: 70ch;    /* 긴 본문/문단의 최대 읽기 폭 */
--layout-gutter: 1rem~2rem; /* 화면 폭에 따른 좌우 여백 */
```

원칙:

- 모든 페이지는 `AppLayout > page__inner`를 통해 같은 본문 레일을 공유한다.
- 홈을 제외한 공개 페이지와 관리자 페이지는 같은 외곽 본문 폭을 사용한다.
- 헤더 내부 폭과 본문 외곽 폭은 같은 기준선에 맞춘다.
- 세로 스크롤바 유무 때문에 페이지별 중앙 정렬이 흔들리면 안 된다. `html`에서 `scrollbar-gutter: stable both-edges`로 양쪽 gutter를 예약한다.
- 글 본문처럼 긴 텍스트는 페이지 전체 폭을 다 쓰지 않고 `70ch` 안팎으로 제한한다.
- 자료 그리드나 방명록 2열처럼 넓은 배치가 필요한 경우에도 외곽 컨테이너 폭은 동일하게 유지한다.
- 페이지별로 `760px`, `1120px` 같은 값을 임의로 섞지 않는다.
- 개별 페이지 컴포넌트에서 외곽 `margin`, `max-width`, `justify-self`를 따로 지정하지 않는다.

## 정보 위계 원칙

- 상단바의 활성 메뉴가 현재 위치를 표시하므로, 공개 페이지 본문 상단에 같은 이름의 큰 제목을 반복하지 않는다.
- 접근성을 위한 `h1`은 유지하되 시각적으로는 숨길 수 있다.
- 본문 첫 화면은 페이지 이름보다 사용자가 바로 할 일이나 볼 내용이 먼저 보여야 한다.
  - 아무 글: 1열 글 목록
  - 자료: 검색/필터와 자료 목록
  - 방명록: 글 남기기 폼과 목록. 단, 메인 콘텐츠와 같은 위계의 큰 메뉴로 두지는 않는다.
  - 검색: 검색 입력창과 결과
- 크기, 색, 여백은 사용자의 다음 행동과 실제 콘텐츠를 우선하도록 배치한다.

## 공통 레이아웃

### 공통 상단바와 도구 영역

홈을 제외한 공개 페이지는 스케치의 배치감을 따르되, 실제 UI에는 제공된 아이콘 이미지를 사용한다.

```txt
[채아무 아이콘 + 그냥 아무거나 올리는 채널]
[아무 글 아이콘+라벨] [자료 아이콘+라벨]          우측 상단: [검색 입력+아이콘] [방명록 벡터 아이콘] [설정 벡터 아이콘]
```

요구사항:

- 왼쪽 브랜드는 홈(`/`)으로 이동하는 링크로 사용한다.
- 데스크톱 브랜드 텍스트는 `그냥 아무거나 올리는 채널` 풀네임을 사용한다.
- 폭이 좁아지면 브랜드 텍스트는 숨기고 아이콘만 표시한다.
- 좁은 화면에서 상단바 메뉴와 도구가 오른쪽으로 잘리면 안 된다.
- 좁은 화면에서는 헤더 본문을 공통 도구 줄 아래로 내려 메뉴와 도구가 겹치지 않게 한다.
- 공통 도구 영역은 모든 페이지에서 fixed 오버레이 안에서 같은 `--layout-page` 레일의 오른쪽 끝을 사용한다. 페이지별/브레이크포인트별로 static/absolute 또는 viewport 직접 `right` 배치로 바꾸지 않는다.
- 평상시 공통 도구 오버레이의 빈 영역은 클릭을 막지 않아야 하며, 검색 폼과 아이콘 버튼만 포인터 이벤트를 받는다. 설정 사이드바는 최상위 전체-뷰포트 오버레이로 열려 클릭/닫기/배경 클릭/Escape 입력을 정상 처리해야 한다.
- 메인 콘텐츠 메뉴는 `아무 글`, `자료`만 큰 정사각형 메뉴로 둔다.
- `방명록`은 메인 콘텐츠와 같은 위계가 아니라 사이트 상단 오른쪽 공통 도구 아이콘으로 둔다.
- 검색은 아이콘만 단독으로 두지 않고 키워드 입력창과 Lucide 계열 SVG 벡터 검색 아이콘을 함께 둔다.
- 방명록/설정 아이콘은 검색 아이콘과 같은 결의 Lucide 계열 SVG 벡터 아이콘으로 그린다. 설정 아이콘은 방문자용 설정 사이드바를 연다.
- 메뉴 박스는 테두리가 있고 둥근 모서리를 가진다.
- 메인 메뉴 버튼은 정사각형으로 통일한다.
- 현재 페이지가 `아무 글` 또는 `자료`일 때는 메인 메뉴 활성 상태로 표시한다.
- 방명록 페이지에서는 우측 상단 방명록 도구 버튼도 활성 색으로 표시한다.
- `/admin/`은 일반 상단바 메뉴에 노출하지 않는다.
- 모바일에서도 모든 메뉴와 도구 접근이 가능해야 한다.
- 공통 상단바는 `src/components/AppLayout.tsx`의 `Header`와 `src/components/SiteTools.tsx`만 수정해서 바꾼다. 개별 페이지에서 상단바/도구를 따로 만들지 않는다.
- `/posts/`, `/archive/`, `/guestbook/`, `/search/`, `/admin/`은 모두 `AppLayout`을 통해 같은 `Header`를 사용해야 한다.

### 홈 우측 도구

홈은 중앙 검색창을 이미 갖고 있으므로 우측 상단 도구에서 검색 입력창은 제외한다.

```txt
우측 상단: [방명록 벡터 아이콘] [설정 벡터 아이콘]
중앙: 로고/채널명, [아무 글] [자료], 검색창
```

### 공통 검색 진입

- 홈의 중앙 검색창은 `/search/?q=검색어`로 이동한다.
- 홈을 제외한 페이지의 우측 상단 검색창도 `/search/?q=검색어`로 이동한다.
- `/archive/` 안에는 자료 전용 검색/필터가 별도로 있다.

### 설정 사이드바

- 설정 아이콘을 누르면 오른쪽 사이드바가 열린다.
- 설정 사이드바는 방문자용 표시 설정이다. `/admin/` 관리자 설정과 혼동하지 않는다.
- 현재 항목:
  - 언어: 한국어 고정
  - 테마: 밝은 테마 고정
- 추후 언어/테마 기능을 실제 동작으로 연결할 수 있게 구조만 먼저 둔다.

### 공통 상태 UI

모든 데이터 페이지는 아래 상태를 표현해야 한다.

- 로딩 중
- 데이터 없음
- 에러 발생
- 재시도 버튼
- 저장 중
- 저장 성공
- 저장 실패

## `/` 홈

### 목적

사이트의 입구. “그냥 아무거나 올리는 채널”이라는 정체성을 보여주고, 아무 글/자료로 이동하게 하며 중앙 검색과 우측 상단 방명록 접근을 지원한다.

### 화면 구성

```txt
중앙 묶음
  - 왼쪽: cha-amu-logo.png 아이콘
  - 오른쪽: 그냥 아무거나 올리는 채널

우측 상단 도구
  - Lucide book-open-text 계열 SVG 벡터 방명록 아이콘 버튼: 방명록
  - Lucide sliders-horizontal 계열 SVG 벡터 설정 아이콘 버튼: 설정 사이드바 열기

주요 메뉴 영역
  - 정사각형 둥근 테두리 박스: posts-icon.png + 아무 글
  - 정사각형 둥근 테두리 박스: archive-icon.png + 자료

검색 영역
  - 둥근 검색창
  - CSS 벡터 돋보기 검색 아이콘
```

### 동작

- 아무글 버튼 → `/posts/`
- 자료 버튼 → `/archive/`
- 우측 상단 방명록 아이콘 → `/guestbook/`
- `/admin`처럼 끝 슬래시 없이 관리자 주소로 들어오면 `/admin/`으로 보정한다.
- 검색어 입력 후 Enter 또는 검색 버튼 → `/search/?q=검색어`

### 모바일

- 로고와 문구는 세로 배치 가능.
- 주요 메뉴 버튼은 1열 또는 2열로 재배치 가능.
- 검색창은 화면 폭 전체를 사용한다.

## `/posts/` 아무 글

### 목적

관리자가 작성한 Markdown 글을 보여준다.

### 화면 구성

```txt
공통 상단바

본문
  - 시각적으로 큰 반복 페이지 제목은 두지 않음
  - 1열 블로그형 글 목록
    - 제목
    - 요약
    - 태그
    - 작성/발행일
  - 선택한 글은 같은 흐름 안에서 아래로 펼쳐서 본문 표시
```

### 데이터

- Google Sheets `posts` 시트에서 `status = published`인 글만 표시한다.
- Markdown 본문은 안전하게 렌더링한다.
- 아무글 안의 이미지는 외부 이미지 URL도 허용한다.

### 동작

- 글 클릭 시 상세를 보여준다.
- URL hash로 특정 글 접근을 지원한다.

```txt
/posts/#post-id
```

- 검색 결과에서 들어오면 해당 글이 바로 보이거나 강조되어야 한다.

### 모바일

- 글 목록은 항상 1열 블로그형으로 둔다.
- 목록과 본문을 데스크톱에서 억지로 2단 분할하지 않는다.
- 본문 폭은 읽기 좋게 제한한다.
- 긴 Markdown/이미지는 화면을 넘치지 않게 처리한다.

## `/guestbook/` 방명록

### 목적

방문자가 익명 이름과 메시지를 남기고, 삭제용 비밀번호로 본인 글을 숨김 처리할 수 있게 한다.

### 화면 구성

```txt
공통 상단바

본문
  - 시각적으로 큰 반복 페이지 제목은 두지 않음
  - 작성 폼
    - 이름
    - 메시지
    - 삭제용 비밀번호
    - Cloudflare Turnstile
    - 작성 버튼
  - 방명록 목록
    - 이름
    - 메시지
    - 작성일
    - 삭제 버튼
```

### 동작

작성:

```txt
이름 + 메시지 + 삭제용 비밀번호 + turnstileToken
→ Apps Script
→ Turnstile 검증
→ 비밀번호 salt/hash 처리
→ Google Sheets guestbook 저장
```

삭제:

```txt
삭제 버튼
→ 삭제용 비밀번호 입력
→ Apps Script 검증
→ status = hidden 처리
```

### 표시 정책

- 공개 목록에는 `status = visible`인 글만 표시한다.
- 삭제/숨김 처리된 글은 공개 목록에 표시하지 않는다.

### 레이아웃

- 방명록은 공개 페이지에서 2단으로 나누지 않는다.
- 작성 폼을 먼저 보여주고, 그 아래에 목록을 이어서 배치한다.
- 아무 글, 방명록, 자료의 본문 외곽 레일은 같은 `--layout-page` 폭을 사용한다.
- 방명록 섹션 전체를 하나의 같은 폭 레일로 묶는다. 작성 폼과 방명록 목록의 가로 너비는 같아야 한다.
- `70ch` 제한은 긴 본문 텍스트에만 적용하고, 페이지/섹션 외곽 폭을 줄이는 용도로 쓰지 않는다.

### 모바일

- 작성 폼은 목록 위에 둔다.
- 입력창은 화면 폭 전체를 사용한다.
- 삭제 버튼은 실수 방지를 위해 확인 단계를 둔다.

## `/archive/` 자료

### 목적

`cha-amu/archive` 이미지 전용 GitHub Pages repo의 manifest를 읽어 자료를 자동 표시하고, 이름/설명/태그로 검색한다.

### 화면 구성

```txt
공통 상단바

본문
  - 시각적으로 큰 반복 페이지 제목은 두지 않음
  - 자료 전용 검색창
  - 태그 필터
  - 정렬 옵션
  - 자료 목록
    - 이미지
    - 이름
    - 설명
    - 태그
```

### 데이터

자료 표시 데이터는 아래를 합쳐 만든다.

```txt
archive/manifest.json
+ Google Sheets assetOverrides
= archive 표시 데이터
```

### 동작

- 페이지 진입 시 manifest와 override를 불러온다.
- `status = hidden`인 자료는 표시하지 않는다.
- 검색 대상:
  - name/displayName
  - description
  - tags
  - imagePath
- 자료 클릭 시 이미지 확대 보기 또는 상세 영역을 표시한다.
- URL hash로 특정 자료 접근을 지원한다.

```txt
/archive/#asset-id
```

### 레이아웃

- 자료는 이미지 중심 탐색이라 기본은 그리드뷰로 둔다.
- 단, 카드형 UI를 고집한다는 뜻이 아니라 썸네일을 빠르게 훑기 위한 배치다.
- 이름/설명/태그 검색과 필터는 그리드 위에 둔다.

### 모바일

- 자료 목록은 화면 폭에 맞춰 1열 또는 2열로 조정한다.
- 이미지 확대 화면은 모바일에서 닫기 버튼이 명확해야 한다.

## `/search/` 통합 검색

### 목적

사이트 전체 검색 결과를 보여준다.

### 검색 대상

- 아무글
- 자료

방명록은 기본적으로 통합 검색 대상에서 제외한다.

### 화면 구성

```txt
공통 상단바

본문
  - 검색 입력창
  - 검색어 표시
  - 결과 개수
  - 결과 그룹
    - 아무글 결과
    - 자료 결과
```

### 주소

```txt
/search/?q=검색어
```

### 동작

- `q` query param을 읽어 검색한다.
- 아무글 데이터와 자료 데이터를 조합해 검색 인덱스를 만든다.
- 결과 클릭 시 해당 페이지 hash로 이동한다.

```txt
아무글 → /posts/#post-id
자료 → /archive/#asset-id
```

### 모바일

- 검색창은 상단에 고정하지 않아도 된다.
- 결과는 1열 목록으로 표시한다.


## 공개 데이터 로딩/캐시 UX

방명록과 아무 글은 GitHub Pages JSON 캐시나 글 작성별 GitHub 커밋을 쓰지 않는다. 브라우저 `localStorage` 캐시를 먼저 보여주고, Apps Script 최신 응답을 뒤에서 병합한다.

### 기본 흐름

```txt
페이지 진입
→ localStorage 캐시가 있으면 즉시 표시
→ 백그라운드에서 Apps Script 최신 목록 요청
→ 서버 응답 도착
→ 서버 목록 기준으로 새 글 추가 / 숨김·삭제 글 제거 / 수정 글 교체
→ localStorage 캐시 갱신
```

### 방명록 작성

```txt
작성 버튼
→ temp id 글을 즉시 화면 상단에 표시
→ Apps Script 저장 요청
→ 성공 시 실제 id 글로 교체
→ 실패 시 temp 글 제거 및 오류 표시
```

### 방명록 삭제/숨김

```txt
삭제/숨김 버튼
→ 화면과 localStorage에서 먼저 제거
→ Apps Script 삭제 비밀번호 검증 요청
→ 실패 시 제거했던 글 복구
```

### 제약

- 완전 첫 방문처럼 localStorage 캐시가 없으면 Apps Script 응답을 기다려야 한다.
- 다른 기기/다른 브라우저의 변경 사항은 백그라운드 최신 목록 요청이 끝난 뒤 반영된다.
- 서버 응답은 실제 공개 상태의 기준이다. 서버 목록에 없는 일반 글/방명록은 숨김 또는 삭제된 것으로 보고 로컬 캐시에서 제거한다.

## `/admin/` 관리자

### 목적

관리자 전용 작업을 수행한다.

### 접근

- 공개 상단바와 홈 메뉴에는 관리자 버튼을 만들지 않는다.
- 관리자는 `/admin/` 주소로 직접 들어가며, 로그인 폼은 `/admin/` 안에서만 보여준다.
- 세션이 없거나 만료되면 로그인 화면을 보여준다.
- 입력/클릭/키보드 활동 중에는 서버 세션을 주기적으로 갱신해 작성 중 만료되지 않게 한다.
- 관리자 글 편집 중인 내용은 브라우저 localStorage에 임시 저장해, 세션 만료나 새로고침 후에도 복구할 수 있게 한다.
- 서버 세션 만료 에러가 발생하면 즉시 로그인 화면으로 되돌리고 재로그인 안내를 표시한다.

### 로그인 화면

```txt
채아무 관리자
  - 관리자 비밀번호 입력
  - 로그인 버튼
  - 오류 메시지
```

로그인 성공 시:

```txt
Apps Script 서명 세션 토큰 발급
→ 브라우저 localStorage 저장
→ 관리자 화면 표시
```

테스트 단계 세션 만료:

```txt
1분
```

운영 단계 세션 만료:

```txt
10시간
```

### 관리자 화면 구성

```txt
상단
  - 관리자 제목
  - 세션 상태
  - 로그아웃 버튼

탭/메뉴
  - 아무글 관리
  - 자료 관리
  - 방명록 관리
  - 설정/로그
```

### 아무글 관리

기능:

- 글 목록 보기
- 새 글 작성
- Markdown 본문 작성
- Markdown 미리보기
- 태그 입력
- published/draft/hidden 상태 변경
- 새 글 기본 상태는 `published`로 두어 저장 직후 `/posts/`에 보이게 한다
- 수정
- 숨김/삭제

UI:

```txt
좌측 또는 상단: 글 목록
우측 또는 하단: 편집 폼
```

모바일에서는 목록과 편집 폼을 위아래로 배치한다. 편집 폼은 React controlled input으로 관리해 글 선택/새 글 전환/Markdown 미리보기가 즉시 화면에 반영되어야 한다. Markdown 입력 영역과 공개 글 미리보기의 본문 폭은 `/posts/` 실제 공개 글 본문 폭(`--layout-readable`)과 같은 기준을 사용한다.

### 자료 관리

이미지 repo에 직접 접근하지 않는다.

기능:

- manifest에서 자동 인덱싱된 자료 목록 확인
- assetOverrides 수정
  - 표시명
  - 설명
  - 태그
  - sourceUrl
  - status
  - sortOrder
- 자료 숨김 처리

하지 않는 것:

- 이미지 업로드
- 이미지 repo commit
- GitHub 토큰 사용

### 방명록 관리

기능:

- 방명록 목록 보기
- visible/hidden/deleted 상태 확인
- 스팸/문제 글 숨김
- 숨김 사유 입력

방명록 원문 삭제보다는 `status = hidden` 처리를 우선한다.

### 설정/로그

기능:

- `settings` 시트의 공개 설정 확인/수정
- `auditLog` 조회
- 관리자 작업 기록 확인

비밀값은 표시하거나 수정하지 않는다.

## 반응형 기준

모바일 우선은 아니지만 아래는 지켜야 한다.

- 360px 폭에서도 주요 기능 사용 가능
- 상단바 메뉴 접근 가능
- 폼 입력 가능
- Turnstile 표시 가능
- 자료 검색 가능
- 관리자 로그인 가능
- 관리자 핵심 기능 사용 가능

권장 breakpoints:

```txt
<= 640px   모바일
641-1024px 태블릿/좁은 화면
> 1024px   데스크톱
```

## 접근성 / 사용성

- 모든 버튼은 텍스트 라벨 또는 aria-label을 가진다.
- 폼 오류는 입력 필드 근처에 표시한다.
- 색만으로 상태를 구분하지 않는다.
- 삭제/숨김 작업은 확인 단계를 둔다.
- 로딩/저장 중에는 중복 제출을 막는다.
- 이미지에는 alt를 제공한다.

## 구현 기본값

구현을 막는 미결정 사항은 없다. 아래 값으로 먼저 구현하고, 실제 화면을 보면서 필요한 경우 문서를 같이 갱신한다.

- 실제 구현 자산 위치는 `public/assets/ui/`로 둔다.
- 관리자 화면은 탭 구조를 기본으로 한다.
- 자료 상세는 목록 안 상세 영역을 기본으로 하고, 이미지 확대는 모달을 사용한다.
- 아무 글 상세는 `/posts/#post-id` 기반으로 같은 1열 흐름 안에서 펼쳐 표시한다.
